# ADR Template

Use this template to propose a new Architecture Decision Record. Each ADR documents a significant decision, the context that motivated it, the option chosen, and the expected consequences.

**Instructions:** Copy this file, rename it to `ADR-NNN-descriptive-name.md`, and fill in all sections. Submit as a Pull Request for discussion.

**Date:** YYYY-MM-DD

*Describe the forces that led to this decision. What problem needs to be solved? What constraints or requirements must be satisfied? Include relevant background, alternative approaches considered, and any discussion that informed the decision.*

## Decision

*Describe the chosen approach in detail. What exactly is being decided? How will it be implemented? Include specific examples, code conventions, naming patterns, or process changes as appropriate.*

## Consequences

*What becomes easier, cheaper, faster, or more reliable as a result of this decision?*

### Negative
