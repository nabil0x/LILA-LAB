# Security Policy

## Reporting a Vulnerability

LILA Lab takes the security of our open-source code seriously. If you discover a security vulnerability in any LILA Lab repository, please report it privately before disclosing it publicly.

**Do not report security vulnerabilities through public GitHub issues, Discord channels, or social media.**

Instead, please email: **lila.lab0x@gmail.com**

You should receive a response within 48 hours. If you don't, follow up via the same channel to ensure we received your report.

## What to Include

- Type of vulnerability (e.g., remote code execution, SQL injection, cross-site scripting)
- Full paths of affected source files
- Steps to reproduce (proof of concept is best)
- Potential impact
- Any suggested remediation (if known)

## Scope

This security policy applies to all repositories under the [LilaLABx](https://github.com/LilaLABx) GitHub organization, including:

- LILA Lab pipeline code and shared libraries
- Infrastructure components (Discord bot, website)
- Dataset tooling and distribution scripts

## Response Process

1. **Acknowledgment** — We'll confirm receipt within 48 hours
2. **Assessment** — We'll determine severity and impact within 5 business days
3. **Fix** — We'll develop and test a fix
4. **Release** — We'll deploy the fix and notify you
5. **Disclosure** — We'll coordinate public disclosure timing with you

## Bug Bounty

LILA Lab is a volunteer-run research collective. We do not currently offer a bug bounty program, but we will acknowledge your contribution in our security advisories and (if desired) in our contributor records.

## Preferred Languages

English is preferred for security reports. Bengali is also accepted.

## Attribution

We maintain a list of security researchers who have responsibly disclosed vulnerabilities. If you'd like to be included, let us know when you submit your report.
